The "1-2 Sales" Daily Routine

Deep Work

2 Hours

  1. Review current master template status (SQLite/Convex sync, Reanimated UI).
  2. Improve one reusable module (e.g., offline lead capture, instant search).
  3. Test deployment speed: clone → brand → run in < 3 days target.
  4. Commit changes to private repo with clear notes.
Show Your Work

30 Minutes

  1. Choose one high-value feature (e.g., 0ms offline sync demo).
  2. Record 60-second screen capture (clean UI, voice-over or captions).
  3. Write short caption + industry tag.
  4. Post to LinkedIn, X, IG/TikTok — include procleaning.expo.app link.
Direct Outreach

1 Hour

  1. Find 5 local businesses (cleaning/roofing/real estate/solar) via Google Maps/Yelp.
  2. Personalize DM/email: mention their industry + pain point (slow apps, bad signal).
  3. Send message with procleaning.expo.app link + 2-min demo offer.
  4. Log contact in CRM (Obsidian note) with follow-up date.

Turn-Key Client Architecture

Phase 1

Ingestion (The Lead)

  1. Receive lead (DM, email, social reply) or send proactive outreach.
  2. Send procleaning.expo.app demo link immediately.
  3. Wait for response (“looks good, can it do X?”).
  4. Reply with branded 1-Page Sales PDF (anchors $1,500 price).
  5. Log interaction in Obsidian CRM note.
Phase 2

Commitment (The Close)

  1. Schedule 15-minute Fit Call (strategy, not sales pitch).
  2. Ask qualifying questions: employee count, iOS/Android mix, current tools.
  3. Send digital contract via DocuSign/HelloSign (define Turn-Key vs Custom scope).
  4. Generate Stripe Checkout link for 50% deposit ($750).
  5. Confirm payment → move note to “Active Projects” folder in Obsidian.
Phase 3

Execution (The Build)

  1. Send onboarding form link (logo vector, hex colors, services list, pricing).
  2. Clone private GitHub Roof-Cleaning-Template into new private repo.
  3. Apply branding (logo, colors) + inject client backend credentials.
  4. Customize logic (3–5 days max) based on paid scope.
  5. Test offline/online sync, Reanimated UI, core features.
  6. Commit changes with clear messages; notify client of progress.
Phase 4

Deployment (The Handover)

  1. Generate TestFlight (iOS) / Expo Internal Testing (Android) link.
  2. Send beta link to client for review.
  3. Receive approval email/text: “Looks good to go”.
  4. Charge remaining 50% ($750) via Stripe.
  5. Publish to App Stores (your dev account or client’s).
  6. Activate $49/mo auto-pay (Managed SaaS only).
  7. Hand over admin credentials + basic usage doc.

End-to-End Client Flow

Fit & Filter
  1. Client inquires or responds to outreach.
  2. Request: GitHub username of tech lead + billing email for cloud.
  3. Determine path: Managed SaaS → standard Turn-Key | Buyout → asset transfer.
  4. Send appropriate contract template.
Asset Transfer (Buyout / HOA)
  1. Code Audit: Remove all personal API keys; convert to ENV variables.
  2. Send Stripe Invoice for Source Code Release Fee.
  3. Wait for full payment clearance.
  4. Transfer GitHub repo ownership to client’s GitHub Org.
  5. Migrate Convex team: invite client as Admin, swap billing card, remove self.
  6. Transfer Expo project to client Org (if applicable).
  7. Send signed Exit Agreement: “No ongoing responsibility post-transfer”.
  8. Archive project in Obsidian + mark complete.
